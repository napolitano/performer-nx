# c64font_dump_ascii

`c64font_dump_ascii` converts binary font files (generated by `glyphtext_to_bin`) back into human-readable ASCII art format.

This tool is useful for:
- Verifying that binary font files were created correctly
- Inspecting individual glyphs after conversion
- Debugging font data corruption
- Exporting for documentation or review

It supports the same **variable glyph dimensions** as `glyphtext_to_bin` (e.g., `5x5`, `8x8`, `12x12`).

## What It Does

- Reads a binary font file produced by `glyphtext_to_bin`
- Interprets binary data according to specified dimensions
- Outputs each glyph as a text block using `*` for set bits and `-` for clear bits
- Validates file size against the glyph dimensions

## Binary File Format

The input binary file is expected to be a dense, fixed-stride table of packed glyphs:

- **Glyph 0** at offset `0 * glyphStride`
- **Glyph 1** at offset `1 * glyphStride`
- **Glyph N** at offset `N * glyphStride`
- ... up to `maxIndex`

Where:
- `bytesPerRow = (width + 7) / 8`
- `glyphStride = bytesPerRow * height`

Missing glyph indices (sparse gaps) are encoded as zero bytes.

## Command Line Interface

```text
c64font_dump_ascii [options] <font.bin>
```

## Build and Run

Compile with a C++17-compliant compiler:

```bash
g++ -std=c++17 -O2 -Wall -Wextra -pedantic \
    resources/c64fontconvert/c64font_dump_ascii.cpp \
    -o resources/c64fontconvert/c64font_dump_ascii
```

### Examples

**Dump an 8x8 binary font file (default):**

```bash
c64font_dump_ascii font8x8.bin
```

**Dump a 5x5 binary font file using CLI option:**

```bash
c64font_dump_ascii --size 5x5 font5x5.bin
```

**Dump a 12x16 binary font file separately specifying width and height:**

```bash
c64font_dump_ascii --width 12 --height 16 font12x16.bin
```

## Output Format

The tool prints:

1. **Summary information:**
   - Input file path
   - Glyph dimensions (WxH)
   - Bytes per row
   - Bytes per glyph
   - Total number of glyphs

2. **Glyph blocks** (one per glyph):
   - Header: `Glyph N:`
   - Height rows of width characters: `*` or `-`
   - Empty line separator between glyphs

## Error Handling

The tool validates and reports:

- **File I/O errors:** missing or unreadable input file
- **File size mismatch:** binary size is not a multiple of `glyphStride`
- **Missing arguments:** required positional argument or option value
- **Invalid dimensions:** width/height out of allowed range
- **Unknown options:** unrecognized CLI flags

All errors are reported with a descriptive message and the usage help is printed.

## Integration with glyphtext_to_bin

The typical workflow:

1. **Create a text glyph file** (optional `Size:` header + `Glyph N:` blocks)
2. **Convert to binary** using `glyphtext_to_bin`:
   ```bash
   glyphtext_to_bin --size 12x12 input.txt output.bin
   ```
3. **Verify the output** using `c64font_dump_ascii`:
   ```bash
   c64font_dump_ascii --size 12x12 output.bin > output.txt
   ```
4. **Compare** the input and output text files to ensure fidelity

## Design Notes

- **No metadata**: The binary file stores only pixel data. Dimensions must be supplied via CLI.
- **Dense layout**: Missing glyphs (sparse gaps) are filled with zeros, creating a dense table.
- **MSB-first order**: Bits are extracted left-to-right within each byte for visual correctness.
