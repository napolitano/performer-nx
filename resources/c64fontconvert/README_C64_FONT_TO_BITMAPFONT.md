# c64font_to_bitmapfont

`c64font_to_bitmapfont` converts binary font files (generated by `glyphtext_to_bin`) into optimized rendering formats suitable for embedded display systems:

- A **C header file** containing packed bitmap data and glyph metrics
- A **BMP preview image** showing all glyphs in a grid

This tool is designed to transform bitmap font data from human-readable or source formats into production-ready rendering tables for embedded systems with tight memory constraints.

## What It Does

1. **Loads** a binary C64-format font file (8x8 glyphs per cell)
2. **Maps** the C64 character set to ASCII (0x10–0x7D)
3. **Derives** missing glyphs through transformations (e.g., backslash from rotated slash)
4. **Packs** glyphs into LSB-first 1-bit-per-pixel format for efficient rendering
5. **Computes** per-glyph metrics (bounding box, advance, offset)
6. **Outputs**:
   - C header with packed bitmap and metrics struct
   - BMP preview atlas for visual verification

## Input Format

The input must be a **binary font file** in the format produced by `glyphtext_to_bin`:

- Each glyph occupies fixed bytes (e.g., 8 bytes for an 8x8 glyph)
- Dense layout: Glyph N at offset `N * bytesPerGlyph`
- MSB-first pixel order within each row

Example workflow:

```bash
glyphtext_to_bin 8x8font.txt font.bin
c64font_to_bitmapfont font.bin output.h output.bmp MyFont
```

## Command Line Interface

```text
c64font_to_bitmapfont <input.bin> <output.h> <output.bmp> <FontName>
                      [firstChar] [lastChar] [lineHeight] [topPadding] [baseline] [blankWidth]
```

### Required Arguments

- `<input.bin>` — Binary font file from `glyphtext_to_bin`
- `<output.h>` — Path to output C header file
- `<output.bmp>` — Path to output BMP preview
- `<FontName>` — Base name for C identifiers (e.g., `MyFont` → `myfont_bitmap`, `MYFONT_H__`)

### Optional Arguments

- `firstChar` — First ASCII code to map (default: `16`)
- `lastChar` — Last ASCII code to map (default: `125`)
- `lineHeight` — Line height for atlas and metrics (default: `10`, min: `8`)
- `topPadding` — Top padding in pixels (default: `1`)
- `baseline` — Baseline offset from top (default: `8`)
- `blankWidth` — Width for space/blank glyphs (default: `4`)

## Output Formats

### C Header File

Example generated header (`myfont.h`):

```c
#ifndef __MYFONT_H__
#define __MYFONT_H__

#include "BitmapFont.h"

static uint8_t myfont_bitmap[] = {
    0x00, 0x18, 0x3c, 0x66, 0x7e, ...,
    // Packed bitmap data (LSB-first 1-bit-per-pixel)
};

static BitmapFontGlyph myfont_glyphs[] = {
    { offset, width, height, xAdvance, xOffset, yOffset },
    // Metrics for each glyph in the character range
};

static BitmapFont myfont = {
    1, myfont_bitmap, myfont_glyphs, 16, 125, 10
};

#endif // __MYFONT_H__
```

**Struct fields:**
- `offset` — Byte offset into `bitmap[]`
- `width` — Visual width (after trimming empty columns)
- `height` — Visual height (typically 8 for 8x8 source)
- `xAdvance` — Cursor advance in pixels
- `xOffset`, `yOffset` — Rendering offsets relative to cursor

### BMP Preview

A 24-bit RGB BMP image (uncompressed) showing:
- All glyphs arranged in a 16×N grid
- Each glyph in an 8×lineHeight cell
- White pixels on black background
- Useful for visual verification of conversions

## Glyph Transformations

The tool creates derived glyphs from the C64 font to complete the ASCII range:

- **0x5C** (backslash) — Horizontal mirror of `/` (0x2F)
- **0x5E** (caret) — 90° clockwise rotation of `<` (0x3C)
- **0x60** (backtick) — Horizontal mirror of `` ´ `` (0x27)

All other characters map directly from the source C64 character set or are left blank (0x10–0x1F).

## Packing Format

Glyphs are packed into **LSB-first 1-bit-per-pixel** format to match the expected rendering pipeline:

- Each glyph row is a sequence of 8-pixel chunks
- Pixels are consumed left-to-right, LSB first within each byte
- Unused trailing bits within a byte are zero-padded
- Example (4-pixel glyph): `pixel0 pixel1 pixel2 pixel3 0 0 0 0` → byte value `pixel3<<3 | pixel2<<2 | pixel1<<1 | pixel0`

This packing enables efficient scanline rendering without bit-level manipulation during display.

## Build and Run

Compile with C++17:

```bash
g++ -std=c++17 -O2 -Wall -Wextra -pedantic \
    resources/c64fontconvert/c64font_to_bitmapfont.cpp \
    -o resources/c64fontconvert/c64font_to_bitmapfont
```

Quick example:

```bash
# Convert 8x8 font and generate outputs
c64font_to_bitmapfont font.bin font.h font.bmp MyFont

# With custom rendering parameters
c64font_to_bitmapfont font.bin font.h font.bmp MyFont 16 125 12 2 10 5
```


## Design Notes

- **Fixed 8x8 source glyphs**: Currently expects input to be exactly 8 bytes per glyph.
- **C64 character mapping**: Hardcoded C64 font layout with special handling for characters 0x10–0x1F (reserved/blank).
- **LSB-first packing**: Chosen to minimize bit-unpacking overhead on embedded rendering engines.
- **Dense output**: All characters in the range [firstChar, lastChar] are serialized, even if blank.

## Integration

After generating the header file, include it in your rendering code:

```c
#include "myfont.h"

// Use myfont struct to render text
canvas_drawString(canvas, x, y, "Hello", &myfont);
```

## Error Handling

The tool reports errors for:

- Missing input file or invalid format
- Incorrect file size (not a multiple of 8 bytes)
- Invalid command-line arguments
- File I/O failures
